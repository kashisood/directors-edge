System design

Scaling

  • Vertical (up): bigger machine. Simple, has a ceiling, single point of failure.
  • Horizontal (out): more machines + load balancer. Scales far, fault-tolerant, needs stateless apps.
  • Rule: scale stateless app tier out; scale DB up first, shard last.

Load balancing

  • Round robin, least-connections, IP hash.
  • Health checks route around dead nodes.
  • Enables zero-downtime deploys + horizontal scale.

Caching

  • Layers: browser → CDN → app memory → Redis → DB.
  • Hard part: invalidation (TTL, write-through, evict on change).
  • Watch: stampede, caching private data.

CAP / consistency

  • On a partition, choose Consistency or Availability.
  • Strong = money/inventory. Eventual = feeds/counts.
  • ACID (relational) vs BASE (NoSQL scale).

BCP vs DR

  • BCP: keep the whole business running (people, vendors, systems).
  • DR: restore IT systems/data. DR lives inside BCP.
  • DR tiers: Backup → Pilot light → Warm standby → Active-active.

RTO & RPO

  • RPO: how much data can we lose (backup frequency).
  • RTO: how long until restored (standby hotness).
  • Both are business decisions → cost. Set per system.

Metrics & reliability

SLI / SLO / SLA

  • SLI: the measured number.
  • SLO: internal target.
  • SLA: contractual promise (penalties).

DORA (delivery health)

  • Deployment frequency
  • Lead time for changes
  • Change failure rate
  • Time to restore service

Latency

  • Use p50/p95/p99, not averages.
  • p99 = slowest 1% (the tail users feel).
  • Latency vs throughput is a trade-off.

Error budget

  • 100% − SLO = allowed unreliability.
  • Spent → stop features, fix reliability.
  • Each extra "nine" costs ~10× more.

Leadership & delivery

Decision types

  • Reversible (2-way door): decide fast, delegate.
  • One-way door: slow down, gather data.
  • Most decisions are reversible.

Iron triangle

  • Scope · Time · Cost — fix two, the third flexes.
  • Push all three → quality silently pays.

Delivery

  • Small, frequent, reversible releases.
  • Canary + blue-green + feature flags.
  • Recovery speed > release perfection.

People

  • Psychological safety = #1 team predictor.
  • Underperformance: diagnose skill/will/fit, support, act.
  • Measure: outcomes > output; growth of the team.

Tech debt

  • Deliberate leverage, not a moral failing.
  • ~15–20% steady allocation; make interest visible.

Metric hygiene

  • Goodhart: a measure made a target gets gamed.
  • Leading vs lagging indicators.
  • Never use metrics as individual scorecards.

Security

AuthN vs AuthZ

  • AuthN: who are you (password, MFA, SSO).
  • AuthZ: what can you do (RBAC, ABAC, scopes).
  • AuthN first, then AuthZ.

OAuth vs OIDC

  • OAuth 2.0 = delegated authorization.
  • OIDC = authentication layer on top of OAuth.
  • "Sign in with Google" uses both.

Secrets

  • Never in source control (rotate if leaked).
  • Use a secrets manager; inject at runtime.
  • Least-privilege keys; auto-rotate; CI secret-scan.

Principles

  • Least privilege · Defense in depth.
  • Secure by default / shift left.
  • Zero Trust: never trust, always verify.

Tech radar snapshot (mid-2026)

Java

  • Current LTS: Java 25 (prev LTS: 21).
  • Virtual threads (21+) = simple high concurrency.
  • Policy: standardize on LTS, stay within one.

Spring Boot

  • Current: Spring Boot 4 on Spring Framework 7.
  • API versioning first-class; AOT faster startup.
  • Spring AI bridges Java to LLMs/RAG.

AI for devs

  • Agentic coding: orchestrate agents, review output.
  • RAG over fine-tuning by default.
  • MCP standard; SLMs; evals + guardrails.

Leader's AI stance

  • Capture leverage, manage risk.
  • Review stays mandatory; no secrets in prompts.
  • Evals before customer-facing AI ships.

Behavioral

STAR

  • Situation · Task · Action · Result.
  • ~20% context, ~60% your actions, ~20% result+learning.
  • Say "I", quantify the result.

Story bank (prep 6–8)

  • Leadership/influence, conflict, failure.
  • Delivery under pressure, growing people, business impact.

Situational skeleton

  • Clarify → name the tension → criteria → decide → follow up.

Traps

  • Weakness: real + how you manage it.
  • Silence: it's OK to stop; offer to go deeper.
  • Composure beats speed.

Say it like a director — one-liners

Trade-offs: "It depends on what we're optimizing for — let me name the trade-off."
Speed: "This is a reversible decision, so I'd rather move now and learn than wait for perfect certainty."
Reliability: "What reliability does this use case actually need, and what does the next nine cost us?"
Scaling: "Keep the compute stateless so scaling and failure both stay boring."
DR: "DR starts with two business numbers — RPO and RTO — not a technology choice."
People: "I measure success by what the team ships without me in the room."
Delivery: "Safety comes from a fast path to production and an even faster path to roll back."
Security: "Make the secure path the easy path, so engineers don't route around it."
AI: "Capture the leverage, manage the risk — review stays mandatory."
Conflict: "Let's run a small spike and let data decide; if it's a one-way door, I'll make the call and explain it."
