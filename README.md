# Director's Edge — Engineering Leadership Interview Prep

A one-stop, static study hub for **Engineering Director / senior software leadership**
interviews, targeting the bar at companies like Google, Microsoft, Amazon, Atlassian, Target, Okta, and Visa.
It's **Google-first**: a deep dedicated Google page is the centerpiece, with Google framing woven into every pillar. It's built to shape how you *think and talk* like a technical leader — not just to hand you answers.

Live site: _enable GitHub Pages and your URL appears here, e.g._ `https://<your-username>.github.io/<repo-name>/`

## Pages

**Google focus**
- **Google Prep** (`google.html`) — the priority deep-dive: hiring committee & the four attributes (GCA, RRK, Leadership, Googleyness), scale system design, behavioral, prep plan, and a Google question bank.

**Start here**
- **Home** (`index.html`) — overview and the three ways in.
- **Prep Path** (`prep-path.html`) — staged plan: Level 1 Foundations → Level 2 Core → Level 3 (FAANG bar), with checklists and gate tests.
- **Cheat Sheet** (`cheat-sheet.html`) — printable one-page quick reference.

**The seven pillars** (each has practice questions tagged **L1 / L2 / L3** with a level filter)
- **Fundamentals**, **System Design** (+ **System Design II**: 30 worked problems & all 30 full deep-dives with architecture diagrams), **Tech Radar** (Java / Spring / AI), **Leadership & Delivery**, **Security**, **Behavioral & Situational**, **Mobile Native** (iOS/Android, Expo, TestFlight).

**Live**
- **This Week** (`this-week.html`) — dynamic digest of the week's top cybersecurity & AI stories. Prefers a weekly `data/news.json` (built by a GitHub Action from RSS feeds, with summaries), falls back to the live Hacker News API in the browser, then a curated list.

**Company prep**
- **Company Prep** (`companies.html`) — Google, Microsoft, Amazon, Atlassian, Target, Okta, Visa + general big-tech patterns.

## Features

Fixed left sidebar, sticky top bar, reading-progress bar, **three reading themes (warm paper default, soft light, dim dark)**, **text-size control**, **per-page reading time**, on-page search,
per-question **level badges + level filter**, scroll-spy contents, and a printable cheat sheet —
all in plain HTML, CSS, and vanilla JavaScript. **No build step, no database, no dependencies.**

> Note: the shared sidebar/top bar are generated by `assets/js/main.js`, so view pages through a
> browser (which is always the case for a hosted site).

## Run locally

```bash
python3 -m http.server 8000   # then visit http://localhost:8000
```

## Host on GitHub Pages (free)

1. Create a new GitHub repository and upload these files (unzip first; include the `assets` folder and `.nojekyll`).
2. **Settings → Pages → Build and deployment → Source: Deploy from a branch**, branch `main`, folder `/ (root)`.
3. Save. After ~1 minute your site is live at `https://<your-username>.github.io/<repo-name>/`.

## Weekly news automation (optional)

The **This Week** page auto-populates. To enable the richer RSS digest with summaries:

1. Keep `scripts/fetch_news.py` and `.github/workflows/news.yml` in the repo.
2. In GitHub: **Settings → Actions → General → Workflow permissions → Read and write**.
3. The workflow runs every Monday (and on-demand via the Actions tab), fetching RSS feeds into `data/news.json` and committing it. The page reads that file; if it's missing it falls back to the live Hacker News feed.
