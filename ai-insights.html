AI Insights: LLMs, RAG, Agents & MCP

The modern AI stack decoded in plain language — what an LLM is, how RAG grounds it in your data, how agents and multi-agent systems take action, what MCP standardizes, and how evals keep it production-grade.

📖 The story

AI moved fast, and the vocabulary got muddy. This page untangles the building blocks — from what a language model actually is, to how you ground it in your data (RAG), to giving it the ability to act (agents), coordinate (multi-agent), and connect to tools in a standard way (MCP).

Read it top to bottom and the buzzwords turn into a clear mental model you can reason and make decisions with.

LLMs — the engine

  • LLM (Large Language Model) — a model trained on vast text that predicts the next token (a word-piece). Everything else is built on this one capability.
  • Tokens — the unit models read/write; you pay per token and latency scales with them. ~4 characters ≈ 1 token.
  • Context window — how much text the model can ‘see’ at once (the prompt + retrieved docs + history). Bigger = more context, more cost.
  • Hallucination — a confident but wrong answer. The core reason we ground models with real data (RAG) and verify with evals.
  • Prompting — how you instruct the model: be clear and specific, give examples, ask for steps, request a format. The cheapest lever before any heavier technique.

RAG — grounding answers in your data

A plain model only knows its training data and can make things up. Retrieval-Augmented Generation fixes that: when a question comes in, you search your own knowledge base and hand the most relevant documents to the model as context, so the answer is current and sourced.

  • Embeddings — turn text into vectors (numbers) that capture meaning, so ‘car’ and ‘automobile’ land near each other.
  • Vector database — stores embeddings and finds the most similar ones fast; the storage layer behind RAG and semantic search.

Prompting vs RAG vs fine-tuning

TechniqueWhat it doesUse when
PromptingInstruct the base model wellAlways — the first and cheapest lever
RAGInject your current data at query timeYou need up-to-date, sourced, company-specific answers
Fine-tuningRetrain the model on examplesYou need a specific style/format or a narrow task at low latency/cost
Rule of thumb

Start with prompting, reach for RAG for knowledge, and fine-tune only when prompting + RAG can’t give the style, format, or cost you need. (More on this on the Tech Radar.)

AI agents & agentic AI

An AI agent is an LLM that can act, not just chat — it’s given tools (APIs, code execution, search) and runs a loop: reason about the goal, take an action, observe the result, and repeat until done. Agentic AI is the broader shift from ‘AI answers questions’ to ‘AI plans and executes multi-step work’ under human direction.

Multi-agent systems

For complex work, one agent is split into several specialized agents coordinated by an orchestrator — e.g. a researcher, a coder, and a reviewer. This sidesteps context-window limits and lets each agent focus, at the cost of more coordination.

MCP — Model Context Protocol

  • MCP is an open standard for connecting AI apps to external tools and data sources in a uniform way — think ‘USB-C for AI tools’.
  • Instead of custom-wiring every integration, a model speaks MCP to any compliant MCP server (your database, a SaaS app, a file store).
  • As organizations add many MCP servers, governing them — access, security, sprawl — becomes a real platform responsibility.
📍 Real example — embeddings & cosine similarity

An embedding turns text into a list of numbers (a vector) so that similar meanings land close together. RAG and semantic search rank results by cosine similarity — the angle between two vectors, from 1 (identical) to 0 (unrelated):

embed("how do I reset my password")  → [0.81, 0.12, 0.55, ...]   (~1536 dims, shown tiny)

compare to stored docs (cosine similarity, 1.0 = identical meaning):
  "steps to recover your account login"   → 0.92  ✓ top hit
  "change account password settings"      → 0.88  ✓
  "update billing card on file"            → 0.41
  "our office holiday schedule"            → 0.09

note: keyword search would MISS "recover your account login" — no shared words,
      but the meaning is close, so the embedding ranks it #1.

What this teaches: embeddings capture meaning, not keywords — “reset password” and “recover account login” share almost no words but sit close in vector space. A RAG pipeline embeds your documents once, stores the vectors in a vector DB (Pinecone, pgvector, FAISS), then at query time embeds the question and returns the nearest chunks to feed the LLM. The quality of retrieval — chunk size, the embedding model, and how many neighbours you pull — usually matters more to answer quality than the LLM choice itself.

Evals & guardrails — making it production-grade

  • Evals — systematic tests of model output (accuracy, safety, format) so you manage quality with data, not vibes. The maturity signal that separates a demo from a product.
  • Guardrails — constraints on what the model can say or do (input/output filters, allowed tools, human approval on high-stakes actions).
  • Human-in-the-loop — keep a person on consequential decisions; let AI draft, humans approve.

🧠 Remember it

Modern AI = LLM + RAG (grounding) + agents (action) + MCP (tools) + evals (quality).

  • RAG grounds answers in your data and cites sources.
  • Agents act via tools in a loop; multi-agent splits work across specialists.
  • MCP standardizes tool/data connections; evals + guardrails make it production-grade.
