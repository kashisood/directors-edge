AI Tools & Modes (incl. Claude)

The same model, many surfaces. A map of the common modes of AI tools — chat, coding agent, API, browser, spreadsheet, desktop automation — and, using Claude as the worked example, who uses each and for what.

📖 The story

The same AI model shows up in many ‘surfaces’ — a chat box, an IDE, a terminal, an API, a browser extension, a spreadsheet. Each surface is tuned for a different job and a different kind of user, and picking the right one matters as much as picking the model.

This page maps the common modes of AI tools, then uses Claude as a concrete example of how one provider's surfaces line up against who uses them and for what.

The common modes of an AI tool

ModeWhat it isMainly for
Chat assistantA conversation UI (web/desktop/mobile)Everyone — Q&A, writing, analysis
Coding agent (IDE/terminal)AI that edits code, runs tests, opens PRsDevelopers
API / platformProgrammatic access to the modelDevelopers building products
Browser agentAI that navigates and acts on web pagesPower users, automation
Spreadsheet agentAI inside a spreadsheetAnalysts, finance, ops
Desktop automationAI that works with your files & tasksNon-technical knowledge workers
Plugins / connectorsAdd tools & data to an assistant (often via MCP)Both — extend any of the above

Claude’s surfaces — a concrete example

Anthropic’s Claude is available through several products. They share the same underlying models but are tuned for different users and jobs:

SurfaceWhat it’s forWho uses it
Claude apps (web, desktop, mobile)General chat assistant — questions, writing, analysis, brainstormingEveryone, technical or not
Cowork (desktop, research preview)Automating file & task work on your computer — documents, spreadsheets, multi-step jobs; supports pluginsNon-developers / business users
Claude Code (CLI)Agentic coding from the terminal — reads/edits your repo, runs commands, opens PRs; supports pluginsDevelopers
Claude Developer Platform / API (api.anthropic.com)Build Claude into your own apps and agents programmaticallyDevelopers / product teams
Claude in Chrome (beta)A browsing agent that navigates and acts on web pagesPower users, automation (beta)
Claude in Excel (beta)A spreadsheet agent inside ExcelAnalysts, finance (beta)

Which model for what

Across those surfaces you typically choose a model by the speed/intelligence/cost trade-off:

ModelBest for
Claude Opus (e.g. Opus 4.8)The hardest reasoning, deep coding, complex agentic work
Claude Sonnet (e.g. Sonnet 4.6)The balanced everyday workhorse — strong and fast
Claude Haiku (e.g. Haiku 4.5)Speed and low cost for high-volume, simpler tasks

How to pick — by person & job

  • Non-technical knowledge worker — start in the Claude app for chat; use Cowork to automate file/task work on your computer.
  • Analyst / finance — the chat app for analysis; Claude in Excel for spreadsheet work.
  • Developer — Claude Code in the terminal for hands-on coding; the API/Platform to embed Claude in your own product.
  • Builder of AI products — the Developer Platform plus MCP connectors and evals (see AI Insights).
  • Anyone extending a tool — plugins/connectors (often MCP) add your data and tools to the assistant.
Keep it current

AI products, model names, and availability (beta vs GA) change frequently. Treat this as a snapshot and verify the latest on the provider’s official docs (for Claude: docs.claude.com and claude.com).

🧠 Remember it

Match the surface to the person and the job, not just the model.

  • Chat app = everyone; Cowork = non-devs automating files/tasks.
  • Claude Code = developers in the terminal; API/Platform = building products.
  • Pick Opus (hardest), Sonnet (balanced), Haiku (fast/cheap) by the task.
